336 Ways to Use OpenClaw (here's what actually works)
RoboNuggets (Jay) Feb 26, 2026 30,670 views Resources (search n38)

Jay scraped 336 use cases for OpenClaw across YouTube, GitHub, Reddit, X, and the open web. OpenClaw is an open-source AI agent you add to messaging apps (Telegram, WhatsApp, Slack) and give its own computer so it acts like a digital employee. It's the fastest-growing open-source project in GitHub history. OpenAI hired the creator, so a mass-market version is likely coming.

Top Categories

Setup & Configuration — The biggest chunk. People perfecting agent workspaces, memory systems, and skills. Jay compares it to early Notion "second brain" mania. Key insight: think in "roles to fill" (admin, developer, marketer) not individual tasks.

Coding & Development — Early adopters skew technical. For serious full-stack work, Claude Code gives more control. OpenClaw's edge is convenience (Jay built a database website from his phone over lunch).

Life Admin & Self-Growth — Where it really shines. Eye drop reminders after LASIK, recycling schedules, email newsletter summaries, meeting prep docs, job search monitoring, voice journaling, custom morning briefings, "learning mode" for curated content discovery.

Smart Home — E-ink displays with historical events, Meta glasses integration. Google/Alexa will likely absorb these.

Health (~3%) — Trackers, allergen detection, fitness coaching. Key limitation: no access to large food databases like MyFitnessPal. Proprietary databases remain a real moat for SaaS.

Content & Marketing — Keyword/trend monitoring, Reddit digests, podcast guest research, Meta ad drafts, competitor research via Meta Ads Library.

Finance (risky) — Bank statement audits, investment alerts, automated trading. Not recommended.

Making Money — Deployment services (SimpleClaw, SetupClaw) are short-term arbitrage. Long-term: managed service providers for AI agents in specific industries, and a "template economy" similar to Notion/Shopify templates.

Key Concepts from Database Screenshots

Model Fallback Chain — Tier 1 (expensive, best quality) > Tier 2 (cheaper) > Tier 3 as last resort. Never get stuck on a single model going down.

Humanizer Skill — Matthew Berman's approach: remove telltale AI patterns. No em dashes, no "certainly," no "delve," no bullet points starting sentences.

Agents as Roles — Brian's mindset shift: don't ask "do I want a personal assistant?" Ask "what roles on my team could this fill?"

PR Review Workflow — Peter Steinberger: first ask the agent "Do you understand the intent of this PR?" before looking at implementation.

Security Council — Matthew Berman: At 3:30 AM, four AI security experts (offensive, defensive, data privacy, operational authenticity) review the entire codebase and Git commits.

Antigravity + Every Image & Video AI is a Cheat Code for Creators (NEW Skill)
RoboNuggets (Jay) Feb 20, 2026 34,536 views Resources (search R56)

A skill that lets Claude Code, Antigravity, Codex, or any agentic platform connect to and prompt any generative AI model. You become the creative director; the agent handles prompting, generation, logging, and delivery.

Five Key Advantages

1. Creative Director Mode — You review and approve. The agent handles image analysis, prompt crafting, Airtable logging, batch generation, and cost tracking.

2. Self-Learning System — Agent trains on your prompt best practices (387+ lines of documentation). Learns from your feedback and external workflows (n8n, etc.). Absorbs new techniques you feed it. Never goes obsolete because integrating new models is just copying API docs.

3. Multi-Agent Parallel Processing — Spawn multiple agents working simultaneously on different products, styles, or models. Three agents generating content in parallel while you step away.

4. No Node Spaghetti — Unlike Weev or n8n workflows that become complex and break with model updates, you just describe what you want. Agent handles polling, waiting, retries, and even automatic failover between providers.

5. Cost Efficiency — Use API aggregators (Key AI, Fal AI, Wavespeed) instead of paying for wrapper platforms like Highsfield. Google AI Studio gives $300 free credit per Gmail account for Nano Banana Pro and VO 3.1.

Workflow Breakdown

1. Provide product photo or brand reference → 2. Agent analyzes image + brand guidelines + prompt best practices → 3. Creates Airtable records with unique prompts and cost estimates → 4. Generates images via chosen models (2 variations each) → 5. You review in Airtable → 6. Select winners, direct agent to animate with video models (Sora 2 Pro, VO 3.1, Kling 3.0) → 7. Agent crafts video prompts with dialogue, camera specs → 8. Videos generate in parallel → 9. All assets logged in Airtable.

The CRT Framework

Jay's organizational structure for setting up your creative AI agent workspace. Works across Claude Code, Antigravity, and Codex.

C — R — T

C

.Claude / .Agent

Your AI agent's brain. A markdown file (CLAUDE.md or .agent) containing all instructions, directives, API key references, provider system configuration, and personality. This is the orchestrator that controls everything.

R

References

Your inputs, brand images, style references, product shots, and prompt best practices documentation (387+ lines). Everything the agent needs as creative fuel. Continuously updated as you refine your style.

T

Tools

Python scripts and workflows that route tasks to different AI models. Handles Airtable updates, image/video generation API calls, cost tracking, and provider failover. Think of each .py file as an n8n workflow in code form.

The Provider System inside CLAUDE.md lets you override which API aggregator handles generation by passing provider="kie", provider="google", or provider="wavespeed" to generation functions.

Blotato: The AI Content Engine

$29/mo subscription for multi-platform content scheduling, caption generation, and transcript extraction. Core tool in your automation stack.

What Blotato Does

  • Auto-generates captions for videos
  • Extracts transcripts from YouTube/podcasts
  • Repurposes content across platforms (Instagram, TikTok, LinkedIn, YouTube)
  • Drafts social copy with hashtags and branding
  • Schedules posts in advance
  • Tracks analytics per platform

Integration with Your Workflow

Step 1: Connect your social media accounts (Instagram, TikTok, YouTube, LinkedIn)

Step 2: Create a brand voice template (brand.md) describing your audience and tone

Step 3: Upload or paste video/podcast content for transcription

Step 4: Have Claude Code or Antigravity call Blotato API to generate captions, summaries, and posts

Step 5: Store results in Airtable for review before publishing

Key Features

  • Smart Clips: Auto-extracts engaging 15-30 second segments from longer videos
  • Caption Variants: Generate multiple caption versions for A/B testing
  • Hashtag Auto-Generation: Suggests trending hashtags per platform
  • Bulk Scheduling: Schedule 30 days of content from a single prompt
  • Analytics Dashboard: Track performance per platform and content type

OpenClaw Setup Guide

Complete step-by-step guide to setting up OpenClaw on a Mac Mini as a fully operational personal AI agent. Based on a real setup building "Mr. Anderson" (Thomas Anderson / Neo).

Version: 1.0 | March 2026

What You Will Have When Done

  • An always-on AI agent running on a dedicated Mac Mini
  • iMessage and Telegram as communication channels
  • Full shell, file system, and browser access
  • Gmail, Calendar, and Google Drive integration
  • GitHub integration with automated workspace backups
  • Web search via Brave Search API
  • Automated daily briefings (morning and evening)
  • A dream routine that consolidates memory overnight
  • Health monitoring, cron job monitoring, and token budget tracking
  • A heartbeat system that checks in proactively throughout the day

Prerequisites

  • Mac Mini (Apple Silicon recommended, M2 or later)
  • macOS 15 (Sequoia) or later
  • Anthropic API key (recommended for instruction following)
  • Telegram account and bot token (from @BotFather)
  • Apple ID for iMessage on the Mini (dedicated bot account recommended)
  • GitHub account with personal access token
  • Google account for the bot (separate from personal Gmail)
  • Brave Search API key (optional but recommended)
  • 2-3 hours of focused setup time
Phase 1: Prepare the Mac Mini
1.1 Disable Sleep
sudo pmset -a sleep 0 disksleep 0 displaysleep 0
sudo pmset -a hibernatemode 0 powernap 0
sudo pmset -a standby 0 autopoweroff 0
sudo pmset -a autorestart 1

Verify with pmset -g. All sleep values should be 0, autorestart should be 1.

Create a caffeinate LaunchAgent as a failsafe:

cat > ~/Library/LaunchAgents/com.openclaw.caffeinate.plist << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>com.openclaw.caffeinate</string>
  <key>ProgramArguments</key>
  <array>
    <string>/usr/bin/caffeinate</string>
    <string>-s</string>
  </array>
  <key>RunAtLoad</key><true/>
  <key>KeepAlive</key><true/>
</dict>
</plist>
EOF

launchctl load ~/Library/LaunchAgents/com.openclaw.caffeinate.plist
1.2 Enable FileVault

System Settings > Privacy & Security > FileVault > Turn On. Store the recovery key safely.

1.3 Firewall + Stealth Mode
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --setglobalstate on
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --setstealthmode on
1.4 Enable Remote Login (SSH)

System Settings > General > Sharing > Remote Login > ON. Restrict to your user only.

1.5 Install Homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

After installation, run the PATH commands Homebrew prints. For Apple Silicon:

echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"

Verify: brew --version

1.6 Install Chrome

Download and install Google Chrome. Set it as default browser: System Settings > Desktop & Dock > Default web browser > Google Chrome.

1.7 Dedicated User Account (Optional)

If this Mac Mini has personal data on it, create a standard (non-admin) user account for OpenClaw. If the Mini is a dedicated blank slate, skip this and run everything from your admin account.

Phase 2: Install OpenClaw
2.1 Run the Installer
curl -fsSL https://openclaw.ai/install.sh | bash
2.2 Run the Onboarding Wizard
openclaw onboard --install-daemon

During the wizard, select:

  • Gateway mode: Local
  • AI Provider: Your choice (Anthropic recommended)
  • Bind address: 127.0.0.1 (loopback only)
  • Messaging channel: Telegram (create a bot via @BotFather first)
  • Skills: Skip entirely on first setup
  • Daemon: Yes, install the LaunchAgent
  • Runtime: Node (not Bun)
2.3 Verify the Bind Address
lsof -iTCP:18789 -sTCP:LISTEN

You want to see localhost:18789. If you see *:18789, the gateway is exposed. Fix it immediately.

openclaw gateway stop

Phase 3: Set Up iMessage (BlueBubbles)
3.1 Install BlueBubbles

Download from bluebubbles.app. macOS will block it initially. Go to System Settings > Privacy & Security and click "Open Anyway."

Grant Full Disk Access when prompted (required for reading the Messages database).

3.2 Configure BlueBubbles
  • Skip Firebase/notification setup (not needed for OpenClaw integration)
  • Skip Private API setup (requires disabling SIP, not worth the tradeoff)
  • Proxy: Select "LAN URL" (not Cloudflare or any tunnel)
  • Set a strong server password
  • Note the LAN URL and port (e.g., http://192.168.1.21:1234)
  • Enable "Keep macOS Awake"
  • Auto Start Method: "Launch Agent (Crash Persistent)"
  • Enable "Start Minimized"
3.3 Wire BlueBubbles to OpenClaw

These values should already be in your config from step 2.4. If not, add them to the channels section:

{
  "bluebubbles": {
    "enabled": true,
    "serverUrl": "http://YOUR_LAN_IP:YOUR_PORT",
    "password": "YOUR_BLUEBUBBLES_PASSWORD",
    "webhookPath": "/bluebubbles-webhook",
    "dmPolicy": "pairing",
    "groupPolicy": "allowlist"
  }
}

In BlueBubbles, add a webhook URL: http://localhost:18789/bluebubbles-webhook?password=YOUR_BLUEBUBBLES_PASSWORD

Restart the gateway: openclaw gateway restart

4.1 Google Workspace (Gmail, Calendar, Drive)

Create a Dedicated Bot Gmail: Create a new Gmail account specifically for the agent (e.g., youragent.bot@gmail.com). Do not use your personal Gmail.

Install gog:

npx clawhub@latest install gog

Set Up Google Cloud OAuth:

  1. Go to console.cloud.google.com, sign in with the bot Gmail
  2. Create a new project (e.g., "OpenClaw")
  3. APIs & Services > Library > Enable: Gmail API, Google Calendar API, Google Drive API

Create a fine-grained Personal Access Token:

Give the token to the agent in chat and tell it to configure Git credentials.

Phase 5: Identity and Operating Documents

Create all workspace files in one pass. These define who the agent is, how it operates, and what it knows.

File Architecture
FilePurpose
IDENTITY.mdFactual: name, role, tools, access, boundaries, execution policy
SOUL.mdPersonality, voice, philosophy, smoke detector protocol
AGENTS.mdWorkflow: communication, memory management, mission control, quality bar
USER.mdAbout the user: who they are, how they think, current phase
BOOT.mdSession startup checklist, recovery protocol
HEARTBEAT.mdProactive check-in cadence and rules
BRIEFING_TEMPLATES.mdNews briefing format, sources, and protocols
MEMORY.mdLong-term curated memory, mission control state

Design Principles:

  • No duplication across files. Each piece of information lives in exactly one place.
  • IDENTITY.md is the operational quick-reference. Facts, access, boundaries.
  • SOUL.md is the personality guide. Voice, philosophy, what good and bad look like.
  • AGENTS.md is the workflow manual. Communication standards, memory management, guardrails.
  • Keep everything lean. Every token in always-loaded files competes with working context.

Key Sections to Include:

IDENTITY.md must contain:

  • Execution Policy: "When the user gives a direct instruction, execute it. Do not queue commands and ask for approval. The only exceptions are the Hard Red Lines."
  • Stakes-based autonomy rules (low/medium/high)
  • Hard Red Lines (never post publicly, never delete critical assets, never spend money, never represent user to third parties, never compromise privacy)

AGENTS.md must contain:

  • Verification Protocol: "After any file write or modification, read the file back and confirm by citing specific content."
  • Filesystem and Git Safety: "Never run git commands at ~/.openclaw/ root. Git operations target ~/.openclaw/workspace/ only."
  • Workspace Hygiene: "After modifying any workspace file, ask whether this content belongs in the system prompt, a skill, or memory."
  • Status Updates: "When a task is complete, confirm it. When it fails, report it. Never go silent on an active task."

Write all files to ~/.openclaw/workspace/ using the cat > file << 'EOF' pattern.

6.1 Create a Private GitHub Repo

Create a new private repo (e.g., "agent-brain"). Keep it empty.

6.2 Initialize the Workspace Repo
cd ~/.openclaw/workspace

cat > .gitignore << 'EOF'

telegram/

*.jsonl
*.log

EOF

git add -A
git commit -m "initial brain snapshot"
git push -u origin main
6.3 Install the Backup Skill

Create ~/.openclaw/workspace/skills/backup/SKILL.md with these critical safety rules:

  • ONLY operate on ~/.openclaw/workspace/
  • NEVER run git commands at ~/.openclaw/ root
  • NEVER commit openclaw.json or exec-approvals.json
  • NEVER use git push --force unless explicitly requested
  • If a push fails, report the error. Do not attempt destructive fixes.
6.4 CRITICAL: Never Git-Track the Parent Directory

The ~/.openclaw/ root directory contains openclaw.json (the gateway config with API keys), exec-approvals.json, credential files, and session data. Running git init here will eventually lead to the config file being overwritten or corrupted during push/pull operations. Only ~/.openclaw/workspace/ should be a Git repository.

Appendix C: Troubleshooting

Gateway won't start / times out:

  1. openclaw gateway stop (wait 5 seconds)
  2. openclaw gateway start
  3. If still failing: tail -30 ~/.openclaw/logs/gateway.log
  4. If the LaunchAgent was unloaded: openclaw gateway install && openclaw gateway start

Agent asks for approval on every command:

  1. Check ~/.openclaw/exec-approvals.json has security: "full" and ask: "off"
  2. Check the Control UI (Nodes > Exec Approvals) shows Full/Off
  3. Restart the gateway after changes

BlueBubbles not receiving messages:

  1. Verify BlueBubbles app is running on the Mini
  2. Verify iMessage is signed in on the Mini
  3. Check the webhook URL in BlueBubbles settings
  4. Restart the gateway

Lessons Learned (Avoid These Mistakes)
  1. Edit the config file once. The original setup required opening ~/.openclaw/openclaw.json over a dozen times. This guide has you build the complete config in one pass.
  2. Never initialize Git at ~/.openclaw/ root. Only the workspace (~/.openclaw/workspace/) should be a Git repo.
  3. Never use git push --force on config directories. The agent may attempt this if a push fails. The operating docs should explicitly forbid it.
  4. Set exec approvals before giving the agent tasks. The default approval settings gate every shell command behind a Terminal prompt.
  5. Don't restart the gateway rapidly. If a restart times out, do a stop, wait 5 seconds, then start.
  6. Verify file operations independently. LLMs will sometimes report success without completing the task. Build a verification protocol into your operating docs.
  7. Install Homebrew and add it to PATH before running the OpenClaw installer. The installer tries to install Homebrew itself but fails because stdin isn't a TTY during piped installs.
